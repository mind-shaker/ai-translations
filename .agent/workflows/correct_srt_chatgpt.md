# Batch Correct SRT via ChatGPT

Цей скілл призначений для автоматичного виправлення транскрибації у великих SRT-файлах через веб-інтерфейс ChatGPT за допомогою `browser_subagent`. Скілл автоматично розбиває текст на батчі, зберігаючи цілісність речень, та перевіряє відповідь на відсутність втрат.

## Алгоритм виконання (для Агента):

### 1. Парсинг та розбиття на батчі
1. Напиши та виконай Python скрипт (наприклад, у `/tmp/prepare_batches.py`), який:
   - Відкриває вказаний користувачем `.srt` файл.
   - Парсить його на окремі сегменти (індекс, таймкод, текст).
   - Формує батчі з сегментів. **Максимальний розмір батчу — 50 сегментів.**
   - Останній сегмент у кожному батчі обов'язково повинен закінчуватися розділовим знаком кінця речення (`.`, `!`, `?`, `...`). Якщо 50-й сегмент не закінчується таким знаком, скрипт повинен брати менше сегментів (наприклад, 48), поки не знайде кінець речення.
   - Зберігає батчі у тимчасові файли або масиви для подальшої обробки Агентом.

### 2. Взаємодія з ChatGPT (для кожного батчу)
Проходь по кожному сформованому батчу по черзі:
1. Запусти `browser_subagent` з командою:
   - Відкрий https://chatgpt.com/
   - **Очисти попередню розмову (почни новий чат)**, щоб уникнути змішування контексту між батчами.
   - Якщо з'явиться CAPTCHA або запит на логін — негайно заверши роботу і повідом користувача (`notify_user`).
   - Відправ наступне повідомлення з текстом поточного батчу:
     ```
     можеш виправити недоліки транскрибації які є в наступному srt файлі і повернути відповідь зберігаючи даний формат srt
     
     [ВМІСТ ПОТОЧНОГО БАТЧУ]
     ```
   - Дочекайся завершення генерації.
   - Витягни лише SRT вміст (відкинь будь-який вступний чи завершальний текст, залишаючи тільки нумеровані блоки). Поверни цей вміст.

2. **Валідація (Через Python скрипт або навички Агента):**
   - Перевір, чи кількість отриманих сегментів збігається з кількістю відправлених.
   - Перевір, чи не змінилися таймкоди (вони мають залишатися ідентичними оригіналу).
   - Якщо валідація **не пройдена**: спробуй обробити цей батч через `browser_subagent` ще 1 раз.
   - Якщо друга спроба знову невдала: **залиши оригінальний батч** без змін і переходь до наступного.

3. Зроби паузу `sleep 3` між викликами сабагента, щоб уникнути обмежень швидкості (Rate Limit).
   
### 3. Фінальна збірка
Коли всі батчі оброблені:
1. Напиши та виконай Python скрипт для об'єднання всіх виправлених (та/або оригінальних) батчів.
2. Перенумеруй усі сегменти послідовно (починаючи з 1).
3. Збережи фінальний результат у файл із суфіксом `_corrected.srt` (наприклад, `video_corrected.srt`).
4. Надай користувачеві звіт:
   - Скільки батчів успішно пройшло перевірку.
   - Скільки батчів довелося залишити оригінальними через помилки валідації.
